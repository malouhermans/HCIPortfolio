Arduino

Om kennis te maken met arduino moesten er 14 opdrachten gemaakt worden. Een aantal van deze opdrachten moesten opgenomen worden in het portfolio. In de onderstaande videos zullen de codes en de werkende arduino in kaart gebracht worden. In de onderstaande tekst zal uigelegd worden hoe elke opdracht werkt.

Opdracht 3b

Voor opdracht 3b moesten er twee LED lampjes om en om gaan knipperen. Op het breadboard heb ik een constructie gebouwd met een aantal draadjes, twee weerstandjes en twee LED lampjes. Het gele LED lampje zit verbonden aan pin 12 op de arduino. Het groene LED lampje zit verbonden aan pin 13 op de arduino. Op het breadboard worden beide LED lampjes met een stroomkring aan elkaar verbonden.

Doordat de LED lampjes gekoppelt zijn aan pin 12 en 13, kunnen de LED lampjes binnen de code een actie meekrijgen. Het gele LED lampje (pin 12) staat in eerste instantie uit. Dit staat in de code aangegeven als;

digitalWrite(12, LOW);

Het woord low geeft aan dat het gele LEDlampje uit zal zijn. Voor het groene LED lampje (pin 13) word een een waarde van high meegegeven. Dit geeft aan dat het groene LED lampje aan moet staan. Binnen de code is dit aangegeven als:

digitalWrite(13, HIGH);

Na deze twee acties word er een delay van 1000ms meegegeven. Dit zorgt ervoor dat er een korte pauze tussen de volgende ahandelingen zal zitten. Na de delay zal er precies het tegen overgestelde plaats gaan vinden. Het gele LED lampje (pin 12) zal eerst aangaan en daarna zal het groene LED lampje (pin 13) aangaan.

Door de bovenstaande code binnen een loop te zetten, zal de code continu worden uitgevoerd. Hierdoor blijven beide LED lampjes om en om knipperen.

Opdracht 4c

Voor opdracht 4c is het de bedoeling dat het gele LED lampje Langzaam uit gaat en tegelijker tijd moet het groene LED lampje langzaam aan gaan. Op de arduino en het breadboard is bijna niks veranderd ten opzichten van opdracht 3. Alleen de pins van de LED lampjes zijn veranderd. Het gele LED lampje is aangesloten op pin 9 en het groene LED lampje is aangesloten op pin 10. Binnen een loop word aangegeven dat de lampjes van een helderheid 0 langzaam naar een helderheid van 256 moet gaan. Dit zorgt ervoor dat het LED lampje langzaam aan gaat. In de code is dit geschreven als;

for (int brightness = 0; brightness < 256; brightness++)

In de for loop maak geef ik aan dat het groene LED lampje start met een helderheid van 0. Dit doe ik door de volgende code mee te geven;

analogWrite(greenLedPin, brightness);

Ook geef in binnen dezelfde for loop aan dat het gele LED lampje precies het tegenovergestelde moet gaan doen, door te starten met een helderheid van 255. Dit doe ik door de volgende code mee te geven;

analogWrite(yellowLedPin, 255 - brightness);

Door deze code binnen een for loop te gebruiken, zullen beide LED lampjes in tegengestelde richting langzaam aan en uit gaan.

Opdracht 6c

Voor opdracht 6c moest er hetzelfde effect hebben als opdracht 4. De LED lampjes moeten om en om langzaam aan en uit gaan, wanneer er aan de POT-meter gedraaid word. Hiervoor moet de stroomkring op het breadboard worden aangepast. Er moeten extra draadjes worden aangesloten om de POT-meter in de stroomkring te krijgen.

Binnen de code word bijna hetzelfde toegepast als in opdrach 4c. Er word alleen rekening gehouden met de POT-meter. De POT-meter is aangesloten op pin A0. Het is echter belangrijk om rekening te houden met de POT-meter, daarom is er overal in de code een aanpassing gemaakt. Er word gewerkt met een sensorValue. Deze sensorValue declareert de waardes van de POT-meter en de waardes van de LED lampjes. Vandaar dat de nieuwe code er zo uit ziet;

void loop(){

sensorValue = analogRead(sensorPin);

sensorValue = map(sensorValue, 0, 1023, 0, 255);

 analogWrite(greenLedPin, sensorValue);

 analogWrite(yellowLedPin, 255 - sensorValue);

 Serial.pintln(sensorValue);

 delay(20);

}

Binnen deze code is de sensorValue erg belanrijk, deze zorgt ervoor dat de helderheid van de LED lampjes gekoppeld worden aan de waardes van de POT-meter. Hierdoor kunnen de LED lampjes tegelijk aan en uit gaan, wanneer er aan de POT-meter gedraait word.

Opdracht 7a

Voor opdracht 7a moest er door middel van een LED matrix een vrolijke en verdrietige smiley getoond worden. Deze twee smileys worden omstebeurt vertoont. Uiteindelijk gaat de LED matrix dus om en om twee verschillende smileys vertonen. Door middel van het breadboard en wat raadjes kunnen de arduino en de LED matrix op elkaar worden aangesloten.

De code werd overgenomen vanuit de opdracht. Binnen de code word er een nieuw object aangemaakt, om de LED matrix aan te kunnen sturen. Er worden een aantal belangrijke events aan het nieuwe object meegegeven rondom de helderheid. Dit is te zien in de void setup.

Binnen de loop word het nieuwe object, LedControl, gedeclareert. Per rij moet aangegeven worden welk lampje aan staat en welke niet. Dit word aangegeven door nullen en enen. Door de enen op een beoaalde plek binnen de code te zetten, kan de vrolijke smiley getoont worden. Tussen de code voor beide smileys, moet een delay staan. Deze delay is nodig om beide smileys na elkaar te vertonen.

Opdracht 8

Voor opdracht 8 moesten er twee LED lampjes gaan branden, wanneer de sensor bedekt word. De lED lampjes moet uit gaan, wanneer er geen licht meer op de sensor valt. Op de arduino is een stroomkring gebouwd met twee LED lampjes, een geel LED lampje en een rood LED lampje. Deze zitten met een aantal draadjes en weerstandjes verbonden aan een lichtsensor. Het gele LED lampje is aangesloten op pin 9 en het rode LED lampje is aangesloten op pin 10. De lichtsensor is aangesloten op pin 0A.

De void setup is niet verandert sinds opdracht 6c. De void loop werkt in principe hetzelfde als 6c, maar de waardes zijn anders. Opdracht 8 vraagt of de seriële monitor geopent kan worden. Zodra deze opent geeft het verschillende waardes. De waardes worden bepaald door de hoeveelheid licht dat de lichtsensor detecteert. De hoogste en laagste waarde moeten worden onthouden.

Hoogste waarde: 310

Laagste waarde: 115

Deze waardes worden ingevuld binnen de code. in de map() functie worden de hoogste en laagste waarde ingevuld. Ook word rekening gehouden met de helderheid van de LED lampjes. De waardes 0 (LED lampjes uit) en 255 (LED lampje volledig aan) worden binnen dezelfde map() functie geplaatst. In de constain functie worden de waarde van de LED lampjes nog een keer meegegeven. Door deze waardes te gebruiken binnen de functies, zullen de LED lampjes gaan reageren op het licht.

Opdracht 9c

Voor opdracht 9c moest er een sirene gemaakt worden. Dit moest aan de hand van een piezo speaker. Door middel van een weerstandje en wat draadjes is er een stroomkring gemaakt met de piezo speaker. De piezo speaker is aangesloten op pin 8.

In de void loop zijn twee for loops geplaatst. De eerste for loop zorgt ervoor dat het geluid oploopt en dus steeds hoger van toon word. De tweede for loop zorgt ervoor dat het geluid daalt en dus steeds lager van toon word. door hier een delay achter te plaatsen, volgen deze geluiden vlak op elkaar, waardoor het als een sirene begint te klinken.

Opdracht 11c

Voor opdracht 11c moest het LED lampje aangaan, wanneer er een object in de buurt van de ultrasonic sensor kwam. Via een aantal draadjes word het gele LED lampje (pin 8) gekoppelt aan de ultrasonic sensor (pin 7 en 8). De code in de video moest gekopieerd worden uit de opdracht. Pin 7 geeft de trigger aan die de sensor actief maakt. Pin 8 geeft de echo aan, dit is de reactie op de trigger.

De code binnen de void loop zorgt ervoor dat de trigger gedecteert word. De trigger binnen die video is mijn hand die in de buurt van de ultrasonic sensor komt.

Binnen het tweede deel van de void loop, word er aangegeven wat er meot gebeuren wanneer er een trigger word gedetecteerd. Er word aangegeven dat het LED lampje aan moet gaan, wanneer er een object binnen een bepaalde range komt. Ook moet het LED lampje uit gaan, wanneer de trigger verdwijnt. Dit zie je gebeuren, wanneer ik mijn hand weer weg haal.

Opdracht 12d

Voor opdracht 12d moest er met arduino en processing gewerkt worden. Er word gebruik gemaakt van de POT-meter. Het is de bedoeling dat de POT-meter gekoppelt word aan de seriële monitor. Door code te gebruiken binnen processing, kan er een schermpje ontstaan waarbinnen een zwarte stip te zien is. Deze stip word groter of kleiner, wanneer er aan de POT-meter gedraaid word. De strip geeft de waardes binnen de seriële monitor aan, maar dan visueler.

De codes voor zowel arduino als processing moesten vanuit de opdracht gekopieerd worden. De processing code en arduino code moesten eerst werken en runnen, voor de seriële monitor geopend mocht worden. Het zou anders niet werken. Binnen de processing code moest de seriële poort worden aangepast. Je kon verschillende getallen gebruiken, bij 3 begon mijn code te werken. Er kon aan de POT-meter gedraaid worden en er ontstond een zwarte stip die groter of kleiner werd.

Opdracht 13b

Voor opdracht 13b was het de bedoeling dat er een rood LED lampje stopte met branden, wanneer er op een micro button geklikt word. Hiervoor moesten er een aantal draadjes en weestandjes verbonden worden met een rood LED lampje en de micro button. Het rode LED lampje is aangesloten op pin 13. De micro button is aangesloten op pin 2.

In de void setup word aangegeven dat de pins bestaan en op het een input of een output is. De input is de micro button en de output is het LED lampje dat uit gaat, wanneer de micro button aangeklikt word.

In de void loop word aangegeven dat het lampje uit moet gaan, wanneer er op de micro button geklikt word. Er word een if/else gebruikt om dit te declareren.

Als,

De button ingedrukt word (HIGH), laat dan het lampje uit gaan (LOW).

Anders,

Laat het lampje aan staan (HIGH).

Door deze code mee te geven weet de arduino, wanneer het lampje wel en niet uitgaat. Het kan heel simpel worden omgedraaid, door de (HIGH) en (LOW) waardes binnen de codes exact om te draaien. Het lampje zal dan aan gaan, wanneer er op de knop gedrukt word.

Opdracht 14b

Voor opdracht 14b moest er gewerkt worden met de servometer. De servometer is met een aantal draadjes aangesloten aan de arduino en breadboard. Binnen de eerste paar opdrachten van opdracht 14 moest de servometer werkend gemaakt worden. Voor opdrach 14b, is het de bedoeling dat er zelf met de code word gespeeld en er dus zelf een rytmisch geluid of iets uit komt.

Ik heb de gekopieerde code van opdracht 14a uitgecomment en vervolgens zelf code beschreven. Ik wou van de for loops af om meer vrijheid te krijgen binnen het ritme. Ik heb het een en ander uitgeprobeerd om de code te snappen. Uiteindelijk heb ik besloten om de waardes naar 0, 80, 100 en 20 te veranderen. Er stond nog een delay van 1000ms achter. Ik besloot deze delays korter te maken door ze allemaal 10ms mee te geven.

Toen ik dit deed begon de servometer enorm hard te trillen. Esther van der Molen en ik hebben hier zo'n tien minuten om gelachen, omdat de servometer op een vibrator begon te lijken. Helaas is er in de beweging niet goed te zien hoe snel de servometer trilt, maar het geluid zegt genoeg. Ik heb besloten om deze servometer-vibrator als eigen ritme voor deze opdracht te gebruiken.

Er is per ongeluk een vibrator gemaakt van de servometer.
